# `platforms/` — repo-local platform hints

This directory ships **empty**. Platform hints are not bundled with the
skill; they live in each consuming repo at:

```
<repo>/.claude/skills/design-coverage/platforms/<auto-name>.md
```

A repo-local hint is generated by the companion `design-coverage-scout`
skill the first time `design-coverage` runs in a repo with no existing
hint. The user reviews the scout's draft and commits the approved hint
to their repo; subsequent runs in the same repo reuse it.

Why hints are not bundled with the skill: a hint named `ios.md` or
`android.md` and shipped at the skill level implies "this is the iOS
hint" / "this is the Android hint". In practice every hint is tuned for
one specific app's idioms (its feature-flag types, its permission
helpers, its naming conventions) and silently degrades on a different
app on the same platform. Generating per-repo hints — and committing
them to the consuming repo, not to this directory — keeps provenance
honest and lets every repo own its own platform knowledge.

Two hints originally shipped here, both tuned for specific apps:

- `ios.md` (extracted from MindBodyPOS PR #5349) — relocated to
  `docs/design-coverage/historical-platform-notes/mindbodypos-ios.md`.
- `android.md` (extracted from Express-Android PR #5190) — relocated to
  `docs/design-coverage/historical-platform-notes/express-android.md`.

Both relocated files are **human reference material only** — neither
the skill nor the scout reads from them.
